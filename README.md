# Next.js ERP Assignment

## 1. 프로젝트 소개

Next.js 기반으로 간단한 ERP 시스템을 구현하는 과제 프로젝트입니다.  
상품 및 거래처 관리, 구매/판매 프로세스, 대시보드 기능을 구현하는 것을 목표로 합니다.

## 2. 기술 스택

#### 프레임워크: Next.js (App Router)

#### 라이브러리: React

#### 언어: TypeScript

- 타입 안정성을 확보하고 코드 가독성을 높이기 위해 사용합니다.

#### 스타일링: Tailwind CSS

- 관리자 페이지 UI를 빠르게 구성하기 위해 사용합니다.

#### 인증: Cookie기반, Next.js Middleware

#### 런타임 환경: Node.js

#### 데이터베이스: SQLite, Prisma ORM

- 로컬 환경에서 쉽게 실행 가능하고 별도 DB 서버 설치가 필요 없기 때문에 선택하였습니다.
- Next.js 환경에서 데이터베이스 모델링과 CRUD 작업을 단순화하기 위해 사용합니다.
- Prisma를 통해 타입 데이터베이스 접근을 안전하게 합니다.

## 3. 프로젝트 구조

```text
/app
  /admin
    /dashboard         // 대시보드 요약
    /partners          // 거래처 CRUD
    /products          // 상품 CRUD
    /purchase-orders   // 구매 처리
    /sales-orders      // 판매 처리
  /api
    /auth
      /login             // 로그인 API
      /logout            // 로그아웃 API
  /login               // 로그인
  /middleware.ts       // 인증 검증 미들웨어
```

비로그인 사용자의 관리자 접근을 방지하기 위해
관리자 영역을 /admin 경로로 통일하고 middleware에서 인증 검사를 수행합니다.

## 4. 주요 기능

### - 관리자 로그인

#### 개발용 계정

```text
ID: admin
Password: admin123
```

#### 인증 흐름

1. 로그인 페이지에서 관리자 계정 입력
2. /api/login API 호출
3. 서버에서 계정 검증
4. 성공 시 인증 쿠키 생성
5. middleware에서 보호 경로 접근 시 쿠키 검사
6. 인증 실패 시 로그인 페이지로 리다이렉트

#### 보호 경로

다음 경로는 인증이 필요한 관리자 영역입니다.

```text
/admin/*
```

### - 상품 관리 (CRUD)

- 상품 등록

- 상품 목록 조회

- 상품 정보 수정

- 안전 재고 설정

- 현재 재고 확인

### - 거래처 관리 (CRUD)

- 거래처 등록

- 거래처 목록 조회

- 거래처 정보 수정

- 거래처 유형 관리 (CUSTOMER / SUPPLIER)

### - 발주 생성 및 입고 처리

- 입고 처리

- 재고 변동 이력 기록

- 현재 재고 자동 반영

### - 주문 생성 및 출고 처리

- 출고 처리

- 재고 변동 이력 기록

- 현재 재고 자동 반영

### - 대시보드 요약 정보

- 상품 수

- 거래처 수

- 재고 현황

- 주요 운영 지표 요약

## 5. 데이터베이스 구조

본 프로젝트는 ERP 시스템의 기본 데이터 구조를 단순화하여 설계하였습니다.

### Product (상품)

상품의 기본 정보를 관리하는 마스터 데이터 테이블입니다.

#### 주요 컬럼

- 상품코드

- 상품명

- 단가

- 안전 재고

- 현재 재고

- 생성일

- 판매여부

안전 재고는 재고 부족 상태를 판단하기 위한 기준값으로 사용됩니다.

### Partner (거래처)

거래처 정보를 관리하는 테이블입니다.

ERP 시스템에서는 고객과 공급사를 모두 거래처로 관리하는 경우가 많기 때문에
본 프로젝트에서는 하나의 테이블에서 거래처 유형으로 구분하도록 설계하였습니다.

#### 거래처 유형

CUSTOMER (고객)

SUPPLIER (공급사)

#### 주요 컬럼

- 거래처명

- 거래처 유형

- 연락처

- 이메일

- 계약 상태

- 생성일

### StockMovement (재고 변동 기록)

상품의 재고 변동을 기록하는 테이블입니다.

재고는 입고 및 출고 이벤트에 의해 변경되며
모든 재고 변경 이력은 해당 테이블에 기록됩니다.

#### 재고 변동 유형

IN (입고)

OUT (출고)

ADJUST (재고 조정)

이를 통해 재고 변경 이력을 추적할 수 있습니다.

## 6. UI 디자인 스케일

관리자 페이지의 일관된 UI를 위해 다음 디자인 스케일을 기준으로 구현합니다.

### 색상 기준

- primary : `#6034A7`
주요 버튼 및 강조 요소

- primary-hover : `#7A45C9`
버튼 hover 상태

- secondary : `#9B6DD6`
보조 강조 색상

- background : `#F0EBFA`
페이지 배경

- surface : `#F8F9FA`
카드 및 패널 배경

- border : `#E2E8F0`
구분선 및 테두리

- text-primary : `#1E1030`
주요 텍스트

- text-secondary : `#64748B`
보조 텍스트

- text-muted : `#94A3B8`
설명 텍스트

- success : `#16A34A`
성공 상태

- warning : `#F59E0B`
경고 상태

- danger : `#DC2626`
오류 상태

### 텍스트 크기 기준

- 36px : 페이지 제목

- 28px : 섹션 제목

- 24px : 카드 제목

- 15px : 기본 텍스트

- 14px : 입력창 / 버튼 텍스트

- 13px : 라벨

- 11px : 보조 설명

- 10px : 배지

### 라운드 값 기준

컴포넌트의 모서리 둥글기 기준입니다.

- 8px : 입력창 

- 8px : 버튼 

- 12px : 카드 

- 16px : 모달 

### 기본 컴포넌트 스타일

#### 입력창

```html
px-4 py-3 rounded-lg text-sm border
```

#### 버튼

```html
px-4 py-3 rounded-lg text-sm font-semibold
```

#### 카드

```html
p-6 rounded-xl border
```

```bash
git clone <repository>
```

2. 패키지 설치

```bash
npm install
```

3. 개발 서버 실행

```bash
npm run dev
```

4. 브라우저 접속

```text
http://localhost:3000
```
